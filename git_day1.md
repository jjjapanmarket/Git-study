# Git学習記録day1

## Gitってなんのために使うのか
- ファイルのバージョンの管理するため

## ターミナルの頻出コマンド
- cdコマンド:ディレクトリを移動します。
- lsコマンド:ディレクトリの内容を表示します。 ls -a コマンドで、隠しファイルを含めたディレクトリ全内容を表示します。
- mkdirコマンド:ディレクトリを新規作成します。
- rmコマンド:ファイルを削除します。
- cpコマンド:ファイルをコピーします。
- mvコマンド:ファイルの移動とファイル名の変更を行います。
- catコマンド:ファイルの中身を表示します。

##　ローカルはワークツリー→ステージ→リポジトリの３つのエリアに分かれている。
- 1. ワークツリー：ファイルを変更する作業場
- 2. ステージ：コミットする変更を準備（追加 git add）
- 3. リポジトリ：スナップショットを記録（コミット　git commit）

## Gitの本質
- データを圧縮して、スナップショットで保存していること

## git init は何のためにやるの？
- 一言で言うと、「ただのフォルダ」を「Gitが管理する特別なプロジェクト」
- 管理開始の合図: git init を実行した瞬間、そのフォルダの中に .git という隠しフォルダが作られます。これが「履歴を保存する倉庫」になる。
- 監視カメラの設置: これ以降、Gitはそのフォルダ内でのファイルの追加や修正をすべて「監視」できるようになる。
