# git_day3学習記録

## GitからGitHubへの完全フロー

PC内の「3つの部屋」から、オンラインの「世界（GitHub）」へ届くまでの全工程。

### 1. [作業机] ワークツリー (Working Tree)
- **やること**: VS Codeでコードを書く、ファイルを消す、修正する。
- **状態**: まだGitは記録していない。

### 2. [玄関の棚] ステージ (Staging Area)
- **コマンド**: `git add .` (エイリアス: `git aa`)
- **役割**: 「この変更を次のセーブポイントに含める」という予約。
- **場所**: PC内の `.git` フォルダ。

### 3. [金庫] ローカルリポジトリ (Local Repository)
- **コマンド**: `git commit -m "メッセージ"` (エイリアス: `git ci`)
- **役割**: PCの中に「セーブポイント」として永久保存する。
- **安心**: ここまで来れば、PC上でファイルを消しても `checkout` で復元可能。

### 4. [世界] リモートリポジトリ (GitHub)
- **コマンド**: `git push origin main`
- **役割**: PCの「金庫」の中身を、インターネット上のGitHubへコピーして同期する。
- **利点**: 他のメンバーと共有でき、PCが壊れてもデータが守られる。

---

## データの流れ図

[ 自分のPC ]                                     [ ネット ]
  作業机  ----(add)---->  玄関  ----(commit)---->  金庫  ----(push)---->  GitHub

# リモートから情報を取得する（フェッチ）　　初めはこっち（理解するまで）
- git fetch <リモート名>

# リモートから情報を取得してマージする（プル）
- git pull<リモート名><ブランチ名>
- githubで編集してpullーVSコードに反映

## GitHubから情報を取得して反映させる（プル）

### コマンド
- `git pull origin main`

### 仕組み（内部で行われていること）
1. `git fetch`（GitHubの最新情報を取ってくる）
2. `git merge`（自分の手元のコードに合体させる）
この2つを同時に実行しているのが `pull` です。

### 使う場面
- GitHub上で直接ファイルを編集した時。
- 別のPCで作業してプッシュした内容を、今のPCに取り込みたい時。
- 【将来】チームメンバーがプッシュした最新のコードを取り込む時。

## リモートの詳細情報を表示する
- git remote show <リモート名>

## リモートを変更・削除する
- 【変更】git remote rename<旧リモート名><新リモート名>
- 【削除】git remote rm <リモート名>
